• The filter operations can be applied to various types of columns:

    • String: All columns of string type support the following operations: "equals", "contains", "like" and "not empty". The "like" operation is similar to SQL's LIKE operator and allows the use of the "%" symbol. For example, if you need to select all records that contain the words "staff" and "roles," the filter expression would be "%staff%roles%".

    • Numeric: Columns of numeric type require the operations "Greater than," "Less than," and "equals".

    • Enum: Columns of enum type only require the "equals" operation. This type has a limited array of values.

    • JSON: Columns of JSON type support the operations "equals", "contains", "like" and "not empty" operations.
      . These operations are applied to specific fields within the JSON. To use them, you should use the notation "field name: value;". For example, to select all records where the field "field" is equal to "value," you would use the "equals" operation with the value "field:value".

  • The available filter options are:

    • Topic name: This option searches through all topics using a given topic name. It is a string type value.

    • Partition count: Searches across topics that have a specified number of partitions. It is a numeric value.

    • Replication factor: Searches across topics that have a specified number of replicas, including the leader. It is a numeric value.

    • Consumer count: Searches across topics that have a specified number of consumers. It is a numeric value.

    • Replication state: Searches across topics that have a specified replication state. It is an enum value.

      It can have one of the following values:

      • "ANY_REPLICATED": Choose topics that have at least one replicated partition.

      • "FULLY_REPLICATED" Choose topics that have all replicated partitions.

      • "UNDER_REPLICATED" Choose topics that have at least one under-replicated partition.

    • Config string: Filters across the topic configuration map. It is a JSON type value.

    • Description: Filters across the description of the topic. It is a string value.

  • All filter clauses are combined using the AND operation.

  • Once you have selected all the appropriate filter clauses, press the button to apply the filters and retrieve the desired results.
