• Filtering data can be performed in three different ways, depending on the type of data: String, Numeric and Enum.

    • String data type. You can use operations such as "equals", "contains", "like" and "not empty".

    • Numeric data type. Available operations include "Greater than", "Less than" and "equals".

    • Enum data type. This is a filtering option with a limited number of values. The only available operation is "equals".

  • Filter options:

    • Group name: This allows you to filter by consumer group name, which is a string value.

    • Storage type: filter by storage type. This option lets you filter by storage type, an Enum value. Two possible options are "KAFKA" and "ZOOKEEPER". In older Kafka versions, information about consumer groups was stored in Zookeeper, but since Kafka 0.10, consumer group lists are stored in Kafka, in "__consumer_offsets" topic.

    • Group state: An Enum value representing the state of the consumer group. Possible states are:

      • Unknown: No information about consumer group state

      • PreparingRebalance: A change requiring partition reassignment is happening, so Kafka is in the middle of rebalancing process.

      • CompletingRebalance: Kafka is still rebalancing the group.

      • Stable: Rebalancing is complete, and consumers are functioning as expected.

      • Dead: The group will be removed from this Kafka node soon. This may be due to inactivity, or the group could be migrating to a different group coordinator.

      • Empty: The group exists but has no members.

    • Topics:: Filtering will be performed across all topics consumed by the given consumer group. The record will be accepted if at least one topic name meets the condition.

    • Topics count: It is a numeric value. This option filters consumer groups based on the number of consumed topics.

    • Members: A string value that can filter consumer groups across a list of the consumer group's members (as MemberMetadata per member ID).

    • Members count: A numeric value that can filter consumer groups based on the number of the consumer group's members.

    • Description: A string value that filters values across the consumer group metadata description field.

  • To add a new clause, select the appropriate filter option, operation, and desired value, then click on a button. You can add more than one condition. After adding all the desired conditions, press the button. All added conditions will be combined using the "AND" operation.

  • To share the page with the current result set, press the button. The relevant link will be copied to your clipboard. Then just paste it and use it as needed.
