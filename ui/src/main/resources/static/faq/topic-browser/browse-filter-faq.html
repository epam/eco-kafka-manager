To apply filters, follow these steps:

  • After the first fetch, the list of columns will be populated with all the fetched columns and headers..

  • You can filter by key, payload field, and message header.

  • The filter supports various matching operations, including "equals", "contains", "startWith", "like" and "not empty". There are also a few specific operations, such as "include", "only" and "exclude" which are used exclusively for tombstones.

  • All chosen filters are combined using the AND operator.

  • If the chosen column is of JSON or Map format, including nested arrays, you can use dotted notation to reference the nested parts.
    For example, "a.b: 123" choose records with {a: {b: 123, c: "example"}}, or {a: [{b: 123, c: "example"},{b:100, c: "example2"}]} values. The supported matching operations for these formats are "equals", "contains", "startWith" and "like".

  • The filter operates only on a set of records within a given timeout. The offsets of these records are indicated in the offset range table. If the set of records does not contain the desired data and the end offset has not been reached yet, simply press the button to search in records with higher offsets.

  • There has ability to filter records based on header values.

    To apply filter across headers choose "header: headerKey" in a column list and use "headerValue" or "headerKey.subHeaderKey: subHeaderValue" notation.

    E.g. to choose a record with header "headerKey" equals "headerValue", you should choose "header: headerKey" in a column list and set "headerValue" in a value input item. If it is necessary to choose records with header {headerKey: {subHeaderKey: headerValue}} you should choose column "header: headerKey" in a column list. As the value you should use "subHeaderKey: subHeaderValue" expression.

    The supported matching operations for header filters are "Equals", "contains", "startWith", "like" and "not empty".

  • Operations "exclude" and "only" operations are used exclusively for tombstone messages:

    • "exclude" operation excludes tombstones from fetched records

    • "only" operation shows only tombstone messages
